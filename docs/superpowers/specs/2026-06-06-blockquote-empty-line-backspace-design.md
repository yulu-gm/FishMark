# 引用块末尾空行 Backspace 行为设计

状态：设计已确认，等待实现计划。

## 背景

`editing-region-structural-model` 分支已经把正文空行和引用块内部结构行纳入同一个结构化编辑方向，但当前引用块末尾仍有一个退格边界没有收口：

- 光标位于引用块内最后一条空引用行时，第一次 Backspace 会暴露原始 `>`；
- 再次 Backspace 会把光标移动到上一行附近，但没有按用户意图删除空行；
- 对用户来说，`>` 和 `> ` 都不是可编辑内容，而是引用块内部的空行表示。

这会破坏 Typora 风格的单栏编辑体验：用户看到的是引用块内的空行，Backspace 应该删除这条空行，而不是先暴露 Markdown 标记。

## 目标

- 在引用块末尾空行上按一次 Backspace 时，直接删除当前空引用行。
- 将 `>` 和 `> ` 都视为引用块内的 empty line，不让裸 `>` 成为中间可见状态。
- 如果当前空引用行上方紧邻的是同一引用块末尾的结构性空引用行，也一并删除。
- 删除后光标落在同一引用块内上一条实际内容行的文本末尾，与正文删除空行规则保持一致。
- 保持 Markdown 文本作为唯一事实来源，不通过渲染层遮盖错误状态。

## 非目标

- 不重写引用块、列表或结构行模型的整体架构。
- 不改变 Markdown 保存格式。
- 不调整普通正文空行、列表 Backspace 或引用块 Enter 的既有语义，除非它们直接共享本次边界。
- 不用 CSS 或 decoration 隐藏裸 `>` 来代替真实文档变更。

## 目标行为

当源文档逻辑上等价于：

```md
> 111
>
> - list1
> - list2
>   - child list
>
> |
```

或最后一行源码是 `>`、`> ` 这两种空引用行之一时，按一次 Backspace 后应得到：

```md
> 111
>
> - list1
> - list2
>   - child list|
```

要求：

- 不出现裸 `>` 的可见中间态；
- 不把光标放到列表 marker 附近；
- 不删除 `child list` 内容；
- 不删除 `111` 和列表之间那条正常结构分隔行，因为它不是当前引用块末尾的连续空行。

## 结构判定

本次修复应沿用 `StructuralLineModel` 的方向，把物理源码行分成可编辑内容行、结构性空行和容器标记行。

引用块空行判定需要明确包含：

- `>`；
- `> `；
- 嵌套引用中只有引用前缀和空白的行，例如 `> > `；
- 同一引用块末尾由结构模型派生出的空引用分隔行。

这些行在 Backspace 场景下都应被视为引用块内 empty line，而不是普通可编辑文本。

## 删除范围

Backspace 的删除范围从当前光标所在空引用行开始，向上检查同一引用块末尾的连续空引用行：

1. 当前行必须是空引用行，且光标位于该行的可见空内容位置。
2. 向上合并紧邻的结构性空引用行。
3. 遇到第一条实际内容行时停止。
4. 只删除这段末尾空引用行，不跨越到更早的内容分隔区域。

这个规则使当前空行和其上方的结构性空行可以一次删除，同时避免误删引用块内部两个真实内容块之间的分隔行。

## 光标落点

删除后，光标落在同一引用块内上一条实际内容行的文本末尾。

在示例中，落点是：

```md
>   - child list|
```

这里的“文本末尾”指用户可见内容末尾，不是 Markdown marker、引用前缀或隐藏装饰的位置。该规则与正文中从空行 Backspace 回到上一条内容行末尾的行为保持一致。

## 实现边界

推荐实现方式：

1. 让结构行模型稳定识别 `>` 与 `> ` 这两类空引用行。
2. 在引用块 Backspace 命令里消费这个结构语义。
3. 计算要删除的连续末尾空引用行范围。
4. 事务删除该范围，并把 selection 设置到上一条实际内容行的文本末尾。
5. 继续复用现有 selection normalization，确保光标不会落入隐藏 marker。

不推荐只在渲染层隐藏裸 `>`，因为那会保留错误的文档状态和错误的第二次 Backspace 行为。

## 测试策略

只补和本次行为直接相关的测试，避免无关测试扩张：

- 结构行模型测试：`>` 和 `> ` 在引用块空行语义上等价。
- Backspace 命令或 renderer 测试：在引用块末尾空引用行按一次 Backspace，删除当前行和紧邻结构空行，光标落到上一条实际内容行文本末尾。
- 编辑体验 probe：覆盖用户截图场景，确保不会出现裸 `>` 中间态。

验证时优先运行相关测试、`npm run test:editing-experience`、`npm run typecheck`、`npm run lint` 和 `npm run build`。当前任务在 macOS 验收，不要求运行 Windows 打包测试。
