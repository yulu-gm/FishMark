# TASK-008 Execution Handoff

改了什么：
- 在 `packages/markdown-engine/src/` 新增 block-map 类型和 `parseBlockMap()` 解析入口
- 使用 `micromark` 的公开 parser 事件流提取顶层 `heading`、`paragraph`、`list`、`blockquote`
- 为 block 输出 deterministic `id`、offset、line range，以及 heading depth / ordered-list 元数据
- 把 `packages/markdown-engine` 纳入 Vitest 与 TypeScript 门禁，并补了针对 parser 的单测

人工验收草稿步骤：
1. 打开 `packages/markdown-engine/src/parse-block-map.test.ts`，核对混合 fixture 是否覆盖 `heading` / `paragraph` / `list` / `blockquote`
2. 确认测试断言里的 `startOffset` / `endOffset` 能通过 `source.slice(...)` 回拿原始 Markdown 文本
3. 确认 parser 不会把 list 或 blockquote 内部 paragraph 泄漏成顶层 block
4. 确认当前实现没有把 block map 接进 renderer，也没有扩张到 active block / block rendering

已知风险或未做项：
- 当前只支持最小 top-level block 集合；code fence、image、table、task list、nested item 仍未实现
- list token 的 ordered 信息来自 `micromark` token type；如果后续需要 list item 级别 metadata，需要在 `TASK-012` 再往下拆
- 目前没有 renderer 侧消费方，真实编辑交互要到后续任务才能验证
