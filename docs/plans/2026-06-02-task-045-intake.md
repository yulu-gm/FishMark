Task: TASK-045 脚注语法

Goal: 支持 Pandoc / GFM 风格脚注引用与定义，让 editor preview、active source restore、HTML export 与 round-trip 共享同一份 parser-owned footnote 语义。

In scope:
- 在 `markdown-engine` 中新增 footnote reference / definition 类型和 document-level definition map。
- 支持 `[^id]` / `[^name]` 引用和 `[^id]: text` 定义，保留绝对 source range。
- 未定义引用、重复定义、malformed 定义必须回退为源码显示，不重写 Markdown。
- editor-core 非激活态渲染脚注引用和定义，active 时恢复原始 Markdown。
- HTML export 输出脚注区域和反向链接，且不改变保存目标、dirty 状态或 Markdown 文本。
- 补 parser、decorations、renderer/export 与 round-trip 回归测试。

Out of scope:
- 不实现多段复杂脚注内容的完整 Pandoc 兼容；首版只覆盖 backlog 验收所需的单定义 / 基础延续语义。
- 不引入第二套 Markdown parser。
- 不改变链接 reference definition 的既有行为，footnote definition 使用独立 map。
- 不实现 Mermaid 或数学公式。

Acceptance:
- `[^id]` 引用和 `[^id]:` 定义可被解析，offset 可回拿原始 Markdown 切片。
- 未定义引用、重复定义、跨段定义不丢失源码，不静默重写文档。
- 脚注引用所在块进入 active 后可直接编辑原始 Markdown。
- HTML 导出包含脚注区域，且不改变保存目标或 dirty 状态。
- IME、undo/redo、round-trip 与现有链接 / 行内格式渲染不回归。
