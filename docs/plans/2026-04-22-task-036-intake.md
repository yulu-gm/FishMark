# TASK-036 Intake

Task: `TASK-036`
Goal: 在当前打开的 Markdown 文件被外部进程修改、删除或移动时，明确阻止 autosave 静默覆盖磁盘变化，并给用户稳定、可恢复的处理路径。

In scope:
- 为当前活动文档路径建立主进程级文件系统监听，并在路径切换时正确重绑/清理
- 把外部变更事件通过 preload 安全桥接到 renderer
- 在 renderer 中引入“外部文件冲突待处理”状态，并在该状态下暂停 autosave 对当前路径的写入
- 提供统一提示 UI，覆盖“重载磁盘版本”“保留当前内存版本”“另存为新文件”三条处理路径
- 覆盖外部修改、外部删除/移动、以及处理后 dirty/autosave 状态恢复的针对性测试

Out of scope:
- 最近文件列表或历史版本能力
- 多窗口同时编辑同一路径时的跨窗口协调
- 崩溃恢复、文件锁或操作系统级冲突合并
- 对未保存新文档（无路径）的外部变化处理
- Playwright / e2e 场景扩充

Landing area:
- `src/main/`
- `src/preload/`
- `src/renderer/`
- 相关单元/集成测试文件
- `docs/test-report.md`
- `docs/progress.md`
- `reports/task-summaries/TASK-036.md`

Acceptance:
- 外部修改当前文件后，应用内会出现明确提示
- 选择“重载”后，编辑器内容与磁盘一致，dirty 状态清除
- 选择“保留当前编辑”后，当前内存内容继续保留，且 autosave 不会在用户未明确决策前默默覆盖外部修改
- 选择“另存为”后，可把当前内存版本保存到新路径，不覆盖外部已变化的原路径
- 文件被删除或移动时会出现明确提示，且不会继续对失效原路径执行 autosave

Verification:
- `npm run test -- src/main/electron-app.test.ts`
- `npm run test -- src/renderer/app.autosave.test.ts`
- `npm run test -- src/renderer/App.test.tsx`
- `npm run lint`
- `npm run typecheck`
- `npm run test`
- `npm run build`
- manual: 运行 `npm run dev`，打开一个已保存文档，在系统外部修改 / 删除该文件，验证提示与三条处理路径

Risks:
- 文件系统监听可能在“应用自己保存”与“外部修改”之间产生竞态，需要明确忽略自身写入还是以文档版本状态兜底
- autosave 与外部冲突状态如果没有统一 gate，很容易在 blur 或 idle timer 中偷偷写回旧内容
- 删除 / 移动在不同平台上的 watcher 事件形态可能不同，需要把 UI 行为建立在归一化事件上
- 冲突提示如果和现有状态条、阅读/编辑模式切换耦合过深，容易引入 UI 回归
