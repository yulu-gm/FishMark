# TASK-010 Heading Rendering Design

## Goal

为标题 block 交付第一版可见的源码/渲染切换：
- 非激活标题保留原始 Markdown 文本，但弱化 `#` 前缀并按层级增强正文视觉
- 激活标题完全回到源码态，允许直接编辑 `#` 和正文
- 不引入替换视图，不破坏 IME、光标映射、选择、undo/redo 和 round-trip

## Chosen Approach

采用 CodeMirror decoration 做“轻渲染”，不做 widget replacement。

具体做法：
- 在 `src/renderer/code-editor.ts` 里继续以现有 `blockMap + activeBlockState + composition guard` 为唯一派生状态链
- 对所有非激活 `heading` block 生成两类 decoration
  - `line decoration`：给整行标题增加按层级区分的渲染态 class
  - `mark decoration`：只包裹开头 `#` 前缀字符，并做弱化显示
- 当前激活的标题不加上述 decoration，因此自动回到源码态

这个方案保留真实文本 DOM，不引入替换层，点击定位和选区映射风险显著低于 widget/replacement。

## Rejected Alternatives

### 1. widget/replacement 渲染整条标题

优点：
- 最像 Typora 的“完全渲染态”

缺点：
- 点击定位、选区恢复、IME composition、撤销重做风险最高
- 对 `TASK-010` 来说过重，也绕开了 `TASK-035` 刚建立的安全边界

结论：
- 暂不采用

### 2. 只调字号字重，不处理 `#`

优点：
- 改动最小

缺点：
- 不能形成明确的源码/渲染切换
- 与 backlog 对标题渲染的目标不匹配

结论：
- 不采用

## Data Flow

现有链路已经是：

`CodeMirror update -> parseBlockMap() -> createActiveBlockStateFromBlockMap() -> onActiveBlockChange()`

`TASK-010` 在同一链路上新增：

`ActiveBlockState -> deriveInactiveHeadingDecorations() -> dispatch decoration effect`

约束：
- decoration 只消费现有 `blockMap` 与 `activeBlockState`
- 不在 React 层重新解析 Markdown
- 不修改 `packages/markdown-engine` 的 block map 结构
- composition 期间冻结 decoration 更新，composition 结束后和 active-block 一起 flush

## Decoration Model

对每个非激活 heading block：
- 在 `block.startOffset` 处挂 `Decoration.line`
- 在 `[block.startOffset, block.startOffset + block.depth)` 范围挂 `Decoration.mark`

这样可以：
- 只弱化 `#` 字符本身，不碰标题正文
- 用 line class 给标题正文设置字号、字重、间距
- 保持真实文本仍然可选、可复制、可点击

CSS contract 采用：
- `.cm-inactive-heading`
- `.cm-inactive-heading-depth-1` 到 `.cm-inactive-heading-depth-6`
- `.cm-inactive-heading-marker`

## Interaction Rules

- 初始光标位于第一个标题内时，该标题保持源码态
- 光标移出标题到段落/列表/引用块后，该标题进入弱渲染态
- 再次点击标题或用键盘移动回标题内时，立即恢复源码态
- `replaceDocument()` 后需要同步重算 active block 和 heading decoration
- 组合输入期间不做 decoration 切换；结束后按最终 selection 和 blockMap flush 一次

## Testing Strategy

自动化重点放在 `src/renderer/code-editor.test.ts`：
- 非激活标题会出现弱化 marker 和 heading line class
- 激活标题时上述 decoration 消失
- 光标在标题与段落之间切换时 decoration 跟随 active block 切换
- `replaceDocument()` 后 decoration 能跟随新文档重算
- composition 期间 decoration 不抖动，结束后只按最终状态 flush

本轮不新增独立 e2e，不扩展 `TASK-022` 的真实输入法矩阵。

## Risks

- decoration effect 如果从 update listener 中重复 dispatch，不处理好会制造无效更新
- `block.startOffset + depth` 只覆盖 `#`，不覆盖后续空格；这是有意选择，避免影响正文映射
- 标题层级样式需要克制，避免把普通编辑器壳提前做成完整排版系统

## Out of Scope

- 段落、列表、引用、代码块或行内格式渲染
- heading marker 完全隐藏
- React 层展示 active block UI
- main/preload/autosave/save 行为变化
