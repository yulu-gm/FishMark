Task: TASK-010
Goal: 交付标题 block 的最小可用源码/渲染切换，让非激活标题更接近渲染态显示，激活标题稳定回到 Markdown 源码态，同时不破坏 IME、光标、选择、undo/redo 和 Markdown round-trip。

In scope:
- 只推进 `TASK-010`，不串做 `TASK-011` 到 `TASK-014`
- 基于 `TASK-009` 已有 active block 状态，只给 heading block 增加显示策略
- 非激活 heading 的 `#` 前缀做弱化或隐藏，正文按标题层级呈现
- 光标进入 heading 时稳定回到源码态，允许直接编辑 `#` 语法和正文
- 复用 `TASK-035` 的 composition guard，避免组合输入期间触发装饰/视图切换
- 为标题场景补自动化回归测试，并记录本轮验证证据

Out of scope:
- 段落、列表、引用块、代码块、行内格式、链接或图片渲染
- 新的 Markdown 解析能力或 block map 结构扩展
- 大纲、搜索、导出、图片导入、崩溃恢复等其他 backlog task
- 完整中文 IME 兼容面扩展；引用块、代码块、多光标、真实跨平台输入法差异仍留给 `TASK-022`
- 改动保存链路、autosave 语义、main/preload 文件访问边界

Landing area:
- `src/renderer/code-editor.ts`
- `src/renderer/code-editor-view.tsx`
- `src/renderer/` 下与编辑器视图相关的样式或视图组件
- `packages/editor-core/` 中仅在需要抽出 heading 显示判定时才增补纯逻辑
- `src/renderer/code-editor.test.ts`
- 视情况补充 `src/renderer/app.autosave.test.ts` 或其他受影响测试

Acceptance:
- 非激活 heading 显示为更接近渲染态的标题，`#` 前缀被弱化或隐藏
- 激活 heading 时可直接编辑原始 Markdown，包含 `#` 前缀
- 鼠标点击、键盘移动或选择进入 heading 时，源码/渲染态切换不出现异常跳光标
- heading 输入、删除前缀、撤销重做后行为仍可预测
- 组合输入期间不丢字、不吞字符、不提前切换装饰
- 不改变磁盘 Markdown 文本保真，不引入保存时重排

Risks:
- 标题装饰直接挂在 CodeMirror 视图层，最容易触发 IME composition 干扰；必须复用 `TASK-035` guard 语义
- 源码态/渲染态切换会影响光标映射、鼠标点击定位和选区边界
- heading 前缀弱化/隐藏如果处理不当，可能破坏撤销重做语义或让用户误判真实文本
- 若把显示策略散落到 React 层，容易绕开 `TASK-009`/`TASK-035` 已建立的编辑器边界
- 当前 IME 基线只覆盖段落、标题、列表三类高频场景，真实 Windows/macOS 输入法差异仍有残余风险
