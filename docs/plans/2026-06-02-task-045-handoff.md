# TASK-045 Handoff

## Scope

TASK-045 implements parser-owned footnote semantics for Pandoc / GFM style references and definitions:

- inline references: `[^id]`, `[^name]`
- definitions: `[^id]: text`
- basic indented continuation lines
- duplicate, undefined, malformed, code-fenced, nested-list, and blockquote-looking syntax fallback to source text

The implementation does not introduce a second Markdown parser and does not change save / dirty behavior.

## Acceptance Notes

- `[^id]` becomes a footnote reference only when the normalized identifier resolves to a unique valid definition.
- Undefined references remain literal Markdown text.
- Duplicate definitions are removed from the semantic definition map and rendered as source text.
- Malformed definitions are rendered as source text.
- Footnote definitions inside code fences, nested list bodies, and blockquotes are not promoted to document-level footnotes.
- Source mode uses the existing TASK-060 view-mode gate, so footnote preview decorations disappear with every other Markdown preview.
- HTML export appends a stable footnotes section with backlinks and leaves malformed / duplicate definitions as plain source lines.
- Independent subagent review returned PASS with no blocking findings; its two non-blocking suggestions were addressed by adding list / blockquote parser and export fallback coverage.

[^note]: Footnote **body**

- inactive WYSIWYM shows `note` as a readable superscript-style reference
- clicking into the paragraph restores `Footnote[^note]` source
- clicking into the definition restores `[^note]: Footnote **body**`
- duplicate and undefined syntax stays visible as Markdown source
- source mode shows all raw Markdown
- exported HTML includes one footnote item and backlinks
