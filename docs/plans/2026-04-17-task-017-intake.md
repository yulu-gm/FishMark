# TASK-017 Intake

Task: `TASK-017`
Goal: Deliver the first stable outline sidebar for heading-based navigation, with the outline floating at the right side of the editor workspace, remaining fixed while the document scrolls, and owning its own independent scroll area.

In scope:
- reuse `parseMarkdownDocument()` heading metadata to derive an outline model from the currently opened Markdown source
- render a minimal outline sidebar in `src/renderer` that lists headings by depth and current document order
- keep the outline pinned on the right side of the editor workspace instead of placing it inside the document scroll flow
- ensure the outline panel can scroll independently when the heading list overflows
- wire outline item clicks to editor navigation so the caret and viewport jump to the corresponding heading
- keep outline content synchronized with document edits and document replacement
- add regression coverage for heading data extraction, outline rendering, and navigation plumbing

Out of scope:
- search, export, recent files, or any other non-outline sidebar features
- collapsing outline sections, drag reordering, context menus, or keyboard-only outline navigation
- changes to `main`, `preload`, file IO bridges, autosave semantics, or Markdown round-trip behavior
- mobile/narrow-screen adaptive redesign beyond keeping the existing shell usable

Acceptance:
- the current document produces an outline list from heading blocks with stable labels and depth
- the outline is displayed on the right side of the editor workspace, outside the document scroll container
- scrolling the Markdown document does not move the outline panel with it
- the outline panel itself scrolls when its content exceeds the available viewport height
- clicking an outline item moves the editor selection and brings the corresponding heading into view
- editing or reopening a document refreshes the outline without breaking autosave or active-block tracking

Risks:
- heading labels must be derived without breaking existing lean `parseBlockMap()` consumers or Markdown round-trip guarantees
- editor jump behavior must preserve cursor mapping and avoid regressions in IME, active-block tracking, or autosave timing
- the right-side panel must not accidentally join the editor scroller; otherwise it will drift with document scroll and fail the requested UX
- shell layout changes must keep the existing desktop canvas stable on both Windows and macOS-sized viewports
