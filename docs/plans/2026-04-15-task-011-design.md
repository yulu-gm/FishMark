# TASK-011 Paragraph Rendering Design

## Goal

为非激活普通段落交付第一版轻量渲染层：
- 非激活段落获得更稳定、更接近阅读态的排版表现
- 激活段落保持完全源码态
- 不干扰选择、输入、IME、undo/redo 和 Markdown round-trip

## Chosen Approach

沿用 `TASK-010` 的 CodeMirror decoration 方案，只对非激活 top-level `paragraph` block 挂 `Decoration.line`，不做 widget replacement，也不隐藏任何字符。

具体做法：
- 在 `src/renderer/code-editor.ts` 里扩展现有 block decoration 派生逻辑
- 对所有非激活 `paragraph` block 增加一组 line class
- 当前激活的 paragraph 不加该 decoration，因此天然保持源码态
- 继续复用 `TASK-035` 建立的 composition guard，避免组合输入期间切 decoration

## Rejected Alternatives

### 1. 用 widget/replacement 把段落替换成阅读态块

优点：
- 视觉上更像 Typora

缺点：
- 点击定位、选区映射、复制、IME 风险过高
- 对 `TASK-011` 来说明显过重

结论：
- 不采用

### 2. 全局增强所有段落，不区分 active/inactive

优点：
- 实现最小

缺点：
- 与 backlog 的“激活块源码态、非激活块偏渲染态”目标不一致

结论：
- 不采用

## Data Flow

沿用当前链路：

`CodeMirror update -> parseBlockMap() -> createActiveBlockStateFromBlockMap() -> derive block decorations`

`TASK-011` 不引入新的解析字段，也不改 React 边界。

约束：
- 段落 decoration 只消费现有 `blockMap` 和 `activeBlockState`
- 与 `TASK-010` 的 heading decoration 共用同一派生面
- composition 期间冻结 decoration 更新，结束后统一 flush

## Decoration Model

对每个非激活 top-level paragraph block：
- 在 `block.startOffset` 处挂 `Decoration.line`

CSS contract：
- `.cm-inactive-paragraph`
- `.cm-inactive-paragraph-leading`

其中：
- `.cm-inactive-paragraph` 负责轻量阅读态排版
- `.cm-inactive-paragraph-leading` 只做更克制的颜色/层次增强，便于与普通源码态区分

段落不引入 mark decoration，因为没有像 heading 那样明确的 marker 需要单独弱化。

## Interaction Rules

- 光标进入 paragraph：该段立即回源码态
- 光标移到其他 top-level block：原段落进入轻量渲染态
- 多段落上下移动时，只切当前激活段
- heading 与 paragraph 共存时，两类 decoration 必须稳定叠加，不互相覆盖
- `replaceDocument()` 后 decoration 需要跟随新文档重算

## Testing Strategy

自动化重点仍放在 `src/renderer/code-editor.test.ts`：
- 非激活 paragraph 有段落渲染 class
- 激活 paragraph 时该 class 消失
- 多段落之间切换稳定
- heading 和 paragraph decoration 能同时存在
- composition 结束时 paragraph decoration 只 flush 一次

## Risks

- 段落没有语法 marker，收益主要来自排版增强，样式不能做得太重
- heading / paragraph decoration 如果各自单独 dispatch，容易互相覆盖；必须在同一 decoration set 中派生
- 真实阅读态仍要留给后续 block rendering 扩展，这一轮只做稳定的基础层

## Out of Scope

- 列表、引用、代码块、链接、行内格式渲染
- 段落文本替换或隐藏 Markdown 字符
- React 层显示 active block UI
- 保存、autosave、main/preload 行为变化
