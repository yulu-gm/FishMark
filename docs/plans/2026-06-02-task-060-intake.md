Task: TASK-060 整篇源码模式

Goal: 在当前 CodeMirror 编辑器中提供窗口级整篇源码模式，让用户通过状态栏 `</>` 按钮在默认 WYSIWYM 与完整 Markdown 源码显示之间切换；切换不得创建第二份文档真相，不得改写 Markdown，不得破坏 selection、scroll、dirty、autosave 或 undo/redo。

In scope:
- 定义 renderer 层窗口级 editor view mode：`wysiwym` / `source`。
- 在状态栏右侧增加 `</>` toggle button，使用 `aria-pressed` 暴露状态。
- 将 view mode 传入 `CodeEditorView` / `CodeEditorController` / `createFishMarkMarkdownExtensions`。
- 在 editor-core decoration 管线中增加 source mode gate：源码模式禁用 inactive block、inline replacement、图片 preview、表格 widget、任务列表 widget、引用 rail、代码高亮和未来预览装饰。
- 源码模式保留 CodeMirror 文本编辑、基础行样式、caret、selection、搜索和历史。
- 切换 view mode 时不调用 `replaceDocument`，不重建 `EditorState`，只通过扩展状态刷新 decorations。
- 多标签同一窗口内共享当前 view mode，新窗口默认 WYSIWYM。
- 补 renderer / editor-core / shell 测试，并同步文档记录。

Out of scope:
- 不持久化 source mode 到 workspace snapshot、tab draft、preferences 或 Markdown 文件。
- 不新增第二个 textarea、第二套编辑器实例或 renderer-side document truth。
- 不实现脚注、数学公式或 Mermaid 预览本身。
- 不改变保存、autosave、HTML export、PDF export 规划或 main/preload IPC。

Acceptance:
- 状态栏显示 `</>` 按钮；有文档打开时点击按钮可在默认 WYSIWYM 与源码模式之间切换。
- 源码模式是窗口级：同一窗口内切换标签后仍保持当前 view mode；新窗口默认 WYSIWYM。
- 标题、链接、图片、表格、任务列表、引用块、代码块、分割线、行内格式在源码模式下全部显示原始 Markdown，不显示 Markdown preview 样式或 widget。
- 从源码模式切回默认 WYSIWYM 后，已有阅读态 / 编辑态和 active block 行为恢复。
- 切换源码模式不改写 `EditorState.doc`，不触发保存，不重置 undo history。
- 切换标签、打开文件、新建文件时不会把某个 tab 的内容同步到另一个 tab。
- HTML export、PDF export 规划和 Markdown 保存继续以原始 Markdown 为输入，不受 view mode 影响。
- IME composition 期间不允许触发会破坏组合输入的装饰重建；若正在 composition，则延后切换或只更新外壳状态。

Risks:
- 触及 decoration 重建、active block、IME guard、selection normalization、table widget、image preview、code highlighting、search 和 workspace 多标签。
- 最大风险是通过重建 EditorState 实现切换，导致 undo/redo、selection 或 dirty/autosave 被重置；实现必须避免。
