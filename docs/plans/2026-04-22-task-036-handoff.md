# TASK-036 Execution Handoff

## 改了什么

- 在 `src/main/external-file-watch-service.ts` 新增按窗口绑定的当前文档 watcher，负责把当前路径的外部修改 / 删除归一化成统一事件。
- 在 `src/main/main.ts` 接入 watcher service，并让 `open` / `open from path` / `save` / `save as` 成功后刷新 watcher 基线；同时新增 `syncWatchedMarkdownFile` IPC 供 renderer 在路径变化时重绑或清空监听。
- 在 `src/preload/preload.ts` 与 `src/renderer/types.d.ts` 暴露 `syncWatchedMarkdownFile()` 和 `onExternalMarkdownFileChanged()` 受限 bridge。
- 在 `src/renderer/document-state.ts` 增加 `externalFileState` 状态机，区分 `idle` / `pending` / `keeping-memory` 三个阶段，并在成功 `open` / `save` / `save as` 后自动清空冲突状态。
- 在 `src/renderer/editor/App.tsx` 中把 autosave 挂到外部冲突 gate 下：冲突存在时取消 pending autosave，不再对原路径静默写盘；用户选择“保留当前编辑”后，后续 `Save` 会自动走 `Save As`，避免覆盖外部修改。
- 在 `src/renderer/styles/app-ui.css` 增加外部文件冲突 banner 与动作按钮样式。
- 补齐 main / preload / renderer / state 层回归测试，并同步更新 backlog、进度、测试用例、测试报告、决策日志与任务总结。

## 人工验收草稿

1. 运行 `npm run dev`，打开一个已有路径的 Markdown 文件。
2. 在应用内修改几行内容，但先不要手动保存。
3. 在系统外部编辑同一路径文件并保存，回到 FishMark，确认出现包含“重载磁盘版本 / 保留当前编辑 / 另存为新文件”的提示。
4. 先点“保留当前编辑”，等待超过 autosave idle delay，确认原文件没有被静默覆盖；再按 `Ctrl/Cmd + S`，确认实际走的是 `Save As`。
5. 重新打开原文件，再在系统外部修改后点“重载磁盘版本”，确认编辑器内容切回磁盘版本，dirty 状态清除。
6. 最后把当前文件在系统外部删除或重命名，回到 FishMark，确认仍有明确提示，不会继续对失效原路径执行 autosave。

## 已知风险与未做项

- 当前 watcher 基于单路径 `fs.watch` + 快照对比，已覆盖本轮“修改 / 删除 / 重绑路径”目标，但未实现更复杂的跨窗口协同或版本合并。
- “保留当前编辑”后的保护策略收敛为“禁止 autosave 覆盖原路径，并把后续 Save 导向 Save As”；如果后续想支持显式“仍然覆盖原文件”，需要单独设计更明确的确认流。
- 本轮尚未补真实桌面壳上的人工验收截图或 Playwright 场景，仍需 acceptance 阶段补一轮桌面操作证据。
