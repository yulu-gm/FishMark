# TASK-004 Save / Save As Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build the minimal save and save-as loop for an already opened Markdown document without breaking the current open-file flow.

**Architecture:** Reuse the `TASK-003` pattern: shared result types in `src/shared/`, all filesystem and dialog work in `src/main/`, a minimal safe bridge in `src/preload/`, and state transitions in `src/renderer/document-state.ts`. Keep the current `<textarea>` as the editing surface and write the current in-memory Markdown text back verbatim.

**Tech Stack:** Electron, React, TypeScript, Vitest
