Task: TASK-046 数学公式语法

Goal: 支持 `$...$` 行内数学公式与 `$$...$$` 块级数学公式，使用 parser-owned AST/range 驱动 editor preview 与 HTML export，并以 KaTeX 按需渲染首版数学公式。

In scope:
- 评估并接入 `micromark-extension-math` / `katex`，记录不选 MathJax / Temml 作为首版默认依赖的原因。
- 在 `markdown-engine` 中新增 inline math 与 block math AST，明确 source range、marker range、content range。
- 明确歧义规则：未闭合 `$`、货币文本、转义、空格、code span / code fence 内 `$`。
- editor-core 非激活态公式预览，active 公式所在行 / 块恢复完整 Markdown 源码。
- KaTeX 渲染失败时 fallback 源码或明确错误展示，不能阻塞输入。
- HTML export 使用同一 math AST 和 KaTeX `renderToString` 路径输出可离线阅读 HTML。
- 补 bundle/perf 预算、parser、decorations、renderer/export、round-trip 测试。

Out of scope:
- 不追求完整 TeX / MathJax 覆盖。
- 不引入 renderer 端第二套 Markdown parser 猜公式范围。
- 不默认选择 MathJax 或 Temml；只在决策日志中保留后续评估入口。
- 不实现脚注或 Mermaid。

Acceptance:
- `$x^2$` 与 `$$\n...\n$$` 可被稳定识别。
- 未闭合 `$`、货币文本和 code span / code fence 内的 `$` 保持普通文本或原代码内容。
- 公式渲染失败时显示源码或明确 fallback，不阻塞编辑器输入；`throwOnError: false` 或等效策略覆盖坏公式。
- active 公式所在行 / 块可直接编辑 Markdown 源码。
- KaTeX 相关 JS / CSS 不进入首屏静态 import 闭包，或在决策日志中记录明确取舍。
- HTML export、round-trip、IME 与 undo/redo 不回归。
